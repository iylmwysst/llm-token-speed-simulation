What Is TPS

What is token per second?

Token per second is the speed at which an LLM streams output. It sounds like an infrastructure metric, but in practice it is one of the biggest drivers of perceived AI quality.

By Pipat Khambun · Updated April 22, 2026

Short answer

A token is a chunk of text used by a language model. “Token per second” measures how many of those chunks the model can send to the user each second while it is generating a response.

The reason this matters is simple: users do not experience “throughput.” They experience waiting, reading, and interruptions. Token speed shapes all three.

Why this metric matters

Teams often focus on first-token latency because it is easy to discuss. But once the model starts streaming, the rest of the experience depends heavily on steady output speed. A low token rate can make a good answer feel hesitant. A bursty stream can make an otherwise fast model feel uneven or unstable.

In chat, token speed affects perceived responsiveness. In coding tools, it changes how long users stare at incomplete thoughts. In writing assistants, it determines whether people can comfortably read as the answer appears.

Low token speed

Often feels deliberate, but can become frustrating on longer answers because users spend too much time waiting for the sentence to complete.

Moderate token speed

Usually feels natural for chat and general assistant use. It gives people enough flow to stay engaged without overwhelming them.

High token speed

Feels impressive, but can make dense output hard to read as it streams. Great for perceived speed, not always ideal for in-place comprehension.

How to interpret token speed correctly

Do not treat tokens per second as a universal quality score. A model that streams at 40 tok/s with consistent pacing may feel better than one that averages 80 tok/s but prints in unpredictable bursts. Smoothness matters. Reading context matters. Output length matters.

The practical question is not “what is the fastest number?” It is “what speed creates the best experience for this product and this kind of answer?”
